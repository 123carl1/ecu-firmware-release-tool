# M1 UI Alpha Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建一个可启动、可演示、不卡 UI 的 PySide6 上位机 Alpha，用 Fake 后端跑通扫描、连接、UDS、E68 刷写和 Trace Log。

**Architecture:** UI 使用 PySide6 `QMainWindow`，所有耗时动作通过 Worker 执行，UI 线程只接收事件并更新控件。Fake 后端保持纯 Python，不依赖 PySide6，后续真实 TSMaster/USB2XXX 后端复用同一接口。

**Tech Stack:** Python 3.11+、PySide6、unittest、现有 `FakeLinAdapter`、`LinDiagTransport`、`FlashWorkflow`、`TraceLogger`。

```python
from unified_can_lin_host_tool.adapters.fake import FakeLinAdapter
from unified_can_lin_host_tool.profile import ToolProfile
from unified_can_lin_host_tool.transport.lin_diag import LinDiagTransport


class FakeHostSession:
    def __init__(self, profile: ToolProfile) -> None:
        self.profile = profile
        self.adapter = FakeLinAdapter(
            responses=[
                (profile.bus.response_id, _lin_single(profile.bus.nad, bytes.fromhex("50 01"))),
                (profile.bus.response_id, _lin_single(profile.bus.nad, bytes.fromhex("50 03"))),
                (
                    profile.bus.response_id,
                    _lin_single(profile.bus.nad, bytes.fromhex("67 01 35 79 24 68")),
                ),
            ]
        )
        self.transport = LinDiagTransport(self.adapter, profile)

    def request_uds(self, payload: bytes) -> bytes:
        return self.transport.request(payload).payload


def _lin_single(nad: int, payload: bytes) -> bytes:
    return bytes([nad, len(payload)]) + payload + bytes([0xFF] * (6 - len(payload)))
```

在 `FakeHostBackend` 中增加：

```python
    def connect(self, channel: UiChannel, profile: ToolProfile) -> FakeHostSession:
        if channel.bus != "LIN":
            raise ValueError("fake backend only supports LIN in M1 Alpha")
        return FakeHostSession(profile)
```

- [ ] **Step 4: 实现 Fake 刷写封装**

在 `FakeHostSession.flash_e68()` 内：

1. 读取 FlashDriver 和 App。
2. 构造 `FakeLinAdapter.for_e68_flash_success()`。
3. 构造 `TraceEventBridge`，其 `write(event)` 同时调用 `TraceLogger.write(event)` 和 `on_event(WorkerEvent(kind="trace", message="LIN frame", trace=event))`。
4. 把 `TraceEventBridge` 传给 `LinDiagTransport(trace_logger=...)`，确保 TX/RX 发生时 UI 能实时收到 trace 事件。
5. 构造 `LinDiagTransport`、`BusSession`、`FlashWorkflow`。
6. 通过内部 `emit(event)` 同时追加到返回事件列表并调用 `on_event(event)`。
7. 返回事件列表，至少包含 `started`、若干 `progress`、若干 `trace`、最终 `result`。

`TraceEventBridge` 建议放在 `fake_backend.py` 内部，保持接口最小：

```python
from collections.abc import Callable

from unified_can_lin_host_tool.core.events import TraceEvent
from unified_can_lin_host_tool.trace import TraceLogger
from unified_can_lin_host_tool.ui.models import WorkerEvent


class TraceEventBridge:
    def __init__(
        self,
        logger: TraceLogger,
        on_event: Callable[[WorkerEvent], None] | None,
    ) -> None:
        self._logger = logger
        self._on_event = on_event

    def write(self, event: TraceEvent) -> None:
        self._logger.write(event)
        if self._on_event is not None:
            self._on_event(WorkerEvent(kind="trace", message="LIN frame", trace=event))
```

不要在这里导入 PySide6。

创建 `ui/workers.py`：

```python
from __future__ import annotations

from PySide6.QtCore import QObject, Signal, Slot

from unified_can_lin_host_tool.backends.fake_backend import FakeHostBackend


class DeviceScanWorker(QObject):
    result = Signal(object)
    failed = Signal(str)
    finished = Signal()

    def __init__(self, backend: FakeHostBackend) -> None:
        super().__init__()
        self._backend = backend

    @Slot()
    def run(self) -> None:
        try:
            self.result.emit(self._backend.scan())
        except Exception as exc:
            self.failed.emit(str(exc))
        finally:
            self.finished.emit()
```
